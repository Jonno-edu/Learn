Lesson 0004: Quaternion Basics

Quaternions store attitude without Euler-angle singularities, but they only behave as rotations when kept near unit length.

The One Thing

A quaternion is four stored values. In this course we use scalar-first layout: 

q = [w, x, y, z]

The identity rotation is [1, 0, 0, 0]. Any attitude quaternion used by the estimator should be normalized after operations that can accumulate numerical error.

Core Operations

OperationMeaningTest idea
normalizeScale quaternion to unit norm.[2,0,0,0] becomes [1,0,0,0].
multiplyCompose rotations.Identity multiplied by any unit quaternion returns the same quaternion.
conjugateNegate vector part.[w,x,y,z] becomes [w,-x,-y,-z].
inverseUndo a unit rotation.For unit quaternions, inverse equals conjugate.
Quaternion multiplication is not commutative. a*b and b*a usually mean different rotations.

Tests should include identity behavior, non-commutativity, and rejection of a zero-norm quaternion during normalization.

Quick Check

Why normalize quaternions in estimator code?

Floating-point integration and multiplication introduce small errors. Normalization keeps the quaternion on the unit sphere so it remains a valid rotation representation.
