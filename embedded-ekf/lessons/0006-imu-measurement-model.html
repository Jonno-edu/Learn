Lesson 0006: IMU Measurement Model

The IMU does not hand us position or attitude. It gives angular rate and specific force, both in the body frame, both with bias and noise.

The One Thing

Prediction starts by correcting raw IMU measurements for estimated bias: 

omega_b = gyro_rad_s - b_g
f_b     = accel_mps2 - b_a

omega_b drives attitude prediction. f_b is rotated into navigation coordinates and combined with gravity during velocity and position prediction.

Specific Force And Gravity

Accelerometers measure specific force, not directly "vehicle acceleration." In this course we use: 

a_n = rotate(q_nb, f_b) + g_n
g_n = [0, 0, 9.80665] m/s^2

For a stationary rocket, a_n should be approximately zero, so the rotated specific force must cancel gravity.

This is why sign conventions matter. In local NED, positive z is down, so gravity is positive in the third component.

C Target

When active, define the IMU sample and bias-correction helpers. 

typedef struct {
    eskf_real_t gyro_rad_s[ESKF_VEC3_DIM];
    eskf_real_t accel_mps2[ESKF_VEC3_DIM];
} ImuSample;

EskfStatus imu_correct_gyro(const ImuSample *imu,
                            const eskf_real_t b_g[ESKF_VEC3_DIM],
                            eskf_real_t omega_b[ESKF_VEC3_DIM]);

EskfStatus imu_correct_accel(const ImuSample *imu,
                             const eskf_real_t b_a[ESKF_VEC3_DIM],
                             eskf_real_t f_b[ESKF_VEC3_DIM]);

Tests should check units, bias subtraction, null pointer handling, and that the helpers do not modify the input sample.

Quick Check

Why do we not add gravity inside the accelerometer correction helper?

Bias correction is a body-frame IMU operation. Gravity is added after rotating specific force into the navigation frame.
