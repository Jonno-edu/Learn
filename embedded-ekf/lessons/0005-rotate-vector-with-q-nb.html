Lesson 0005: Rotate A Vector With q_nb

The estimator must rotate body-frame IMU quantities into the local NED navigation frame. This lesson makes the direction of that rotation explicit.

The One Thing

In this course, q_nb means body to navigation. If v_b is a vector expressed in the rocket body frame, then: 

v_n = rotate(q_nb, v_b)

The name is part of the safety contract. A function that silently rotates the opposite way can produce plausible numbers with the wrong physics.

Quaternion Vector Rotation

One common implementation embeds the vector as a pure quaternion and multiplies: 

v_quat = [0, vx, vy, vz]
rotated = q * v_quat * inverse(q)
out = [rotated.x, rotated.y, rotated.z]

Later, we can optimize this. For the first implementation, readability and testability matter more than saving multiplications.

EskfStatus quat_rotate_vec3(const eskf_real_t q_nb[ESKF_QUAT_DIM],
                            const eskf_real_t v_b[ESKF_VEC3_DIM],
                            eskf_real_t v_n[ESKF_VEC3_DIM]);

The function should reject null pointers and should assume the caller provides a normalized quaternion, or explicitly normalize a local copy if the lesson chooses that contract.

Quick Check

Why does the function name include both quaternion and vector direction?

Because frame direction is part of correctness. q_nb rotates body vectors into navigation coordinates; the inverse rotation would answer a different question.
