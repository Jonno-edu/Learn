Lesson 0003: Vector And Matrix Groundwork

The estimator will use tiny fixed-size vector and matrix operations. This lesson builds the mental model before any Kalman equations appear.

The One Thing

A rocket navigation vector is not just three numbers. It is three numbers in a named frame, with named units. The C helper should be small enough that you can inspect every operation and every loop bound.

We prefer vec3_add(), vec3_scale(), vec3_cross(), and skew3() over a runtime-sized generic matrix object.

The Skew Matrix

The skew matrix turns a cross product into a matrix multiplication. For a vector a = [ax, ay, az]: 

skew(a) = [  0  -az   ay
            az    0  -ax
           -ay   ax    0 ]

It is useful because attitude error terms often appear as skew(vector) * dtheta in linearized equations.

C Target

When active, create include/eskf/eskf_vec3.h and src/eskf_vec3.c.

FunctionContract
vec3_zero(out)Write [0, 0, 0].
vec3_add(a, b, out)Write component-wise sum.
vec3_scale(a, s, out)Write s * a.
vec3_cross(a, b, out)Write a x b.
skew3(a, out)Write the 3x3 skew matrix for a.

Tests should use fixed examples like [1, 2, 3] x [4, 5, 6] and should check null arguments are rejected if the function returns a status.

Quick Check

Why avoid runtime-sized matrix code this early?

The ESKF is dimension-sensitive. Fixed-size helpers make the intended dimensions visible and keep loop bounds reviewable in C.
