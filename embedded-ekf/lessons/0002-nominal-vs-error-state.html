Lesson 0002: Nominal State vs Error State

The ESKF carries two ideas at once: a normal physical state used for prediction, and a small error state used for uncertainty and correction.

The One Thing

The nominal state is the estimator's best current navigation answer. The error state is the small correction we estimate around that answer. The filter covariance belongs to the error state, not directly to the nominal quaternion storage.

In this course, P is 15x15 because it describes [dp, dv, dtheta, dbg, dba], not the 16 stored nominal values.

Nominal State

FieldMeaningFrameUnits
p_n[3]PositionLocal NEDm
v_n[3]VelocityLocal NEDm/s
q_nb[4]Body to navigation attitudeMaps body to NEDunitless
b_g[3]Gyro biasBodyrad/s
b_a[3]Accelerometer biasBodym/s^2

Error State

FieldMeaningDimension
dp[3]Small position correction in local NED3
dv[3]Small velocity correction in local NED3
dtheta[3]Small attitude correction3
dbg[3]Small gyro-bias correction3
dba[3]Small accelerometer-bias correction3

The correction dtheta is not another quaternion. It is a small rotation vector that will later be converted into a small quaternion during error injection.

C Target

When this lesson becomes active, create the first state structs in include/eskf/eskf_state.h. 

typedef struct {
    eskf_real_t p_n[ESKF_VEC3_DIM];
    eskf_real_t v_n[ESKF_VEC3_DIM];
    eskf_real_t q_nb[ESKF_QUAT_DIM];
    eskf_real_t b_g[ESKF_VEC3_DIM];
    eskf_real_t b_a[ESKF_VEC3_DIM];
} EskfNominalState;

typedef struct {
    EskfNominalState nominal;
    eskf_real_t P[ESKF_ERROR_STATE_DIM][ESKF_ERROR_STATE_DIM];
} Eskf;

The first tests should check that initialization gives zero position, zero velocity, identity quaternion, zero biases, and a finite covariance diagonal from config.

Quick Check

Why does the covariance not use the 16 nominal-state dimension?

Because covariance tracks the small error vector. The quaternion is stored with four values, but attitude uncertainty is represented as a three-value small-angle error.
