Packet Parser Patterns

Packet parsers are where C fundamentals become flight-software reliability. The habit is to prove every length and index before reading from the frame or copying into an output struct.

Example Frame

Offset Field Meaning
00xa5Start byte 0.
10x5aStart byte 1.
2msg_idMessage type.
3payload_lenNumber of payload bytes.
4..payloadPayload bytes.
lastchecksumByte-sum of msg_id, payload_len, and payload.

Parser Order

  1. Reject null pointers.
  2. Check minimum frame length before indexing fixed header bytes.
  3. Check start bytes.
  4. Read payload length.
  5. Reject payload lengths larger than the output buffer.
  6. Check exact frame length before reading payload or checksum.
  7. Verify checksum.
  8. Only then write the output packet fields.

The core review question is: "What proves this index is valid?" If the proof is not visible nearby, the parser is not ready for flight code.
