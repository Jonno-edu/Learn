ISR To Task Event Handoff

A reference for moving flight events from interrupt context into one task-owned state machine.

Default Shape

/* ISR side: capture and signal. */
(void)event_bridge_post_from_isr(&bridge, FLIGHT_EVENT_APOGEE_DETECTED);

/* Task side: consume and dispatch. */
FlightEvent event;
if (event_bridge_take_for_task(&bridge, &event))
{
    (void)flight_fsm_dispatch(&fsm, event);
}

Rules

  • One task owns the FSM context.
  • ISR code posts small named events only.
  • Do not block, allocate, or run transition policy in an ISR.
  • Make the full/overflow policy explicit and testable.
  • Use FreeRTOS FromISR APIs when using real kernel objects from interrupt context.
  • Use a queue when event values must be buffered in order; use a task notification when a compact signal is enough.

If multiple execution contexts write the same FSM state, stop and define ownership before adding more RTOS primitives.
