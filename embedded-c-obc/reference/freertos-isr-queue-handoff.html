FreeRTOS ISR Queue Handoff

Use this when interrupt context must deliver one item to task context without running the task's policy in the ISR.

Pattern

BaseType_t higher_priority_task_woken = pdFALSE;

BaseType_t sent = xQueueSendFromISR(queue,
                                    &sample,
                                    &higher_priority_task_woken);

if (sent != pdPASS)
{
    isr_drop_count++;
}

portYIELD_FROM_ISR(higher_priority_task_woken);

Checklist

  • Use FromISR APIs in interrupt context.
  • Initialize xHigherPriorityTaskWoken to pdFALSE.
  • Pass the address of that wake variable to the API.
  • Check for pdPASS before counting a successful send.
  • Keep ISR work short: acknowledge, copy, count simple failures, yield if needed.
  • Let the task receive, validate, and dispatch the flight logic.

Boundary Table

ConcernOwner
Hardware interrupt timingISR
Ordered sample bufferingQueue
Sample validation and FSM dispatchFlight task plus plain C modules
Persistent overload policyHealth module or task-owned runtime state

Do not call blocking APIs, drain queues, or dispatch the flight FSM inside the ISR.
