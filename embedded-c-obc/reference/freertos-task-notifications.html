FreeRTOS Task Notifications

A reference for compact direct-to-task signaling.

Bit Pattern

#define NOTIFY_APOGEE (1u << 2)

/* Producer side. */
post_notification_bits(NOTIFY_APOGEE);

/* Task side. */
uint32_t bits;
if (wait_notification_bits(&bits))
{
    if ((bits & NOTIFY_APOGEE) != 0u)
    {
        dispatch_apogee_event();
    }
}

Rules

  • Use notifications when one known task should be signaled.
  • Use bit notifications when each bit means work is pending.
  • Remember that repeated identical bits coalesce.
  • Use a queue when every item or order matters.
  • Use the FreeRTOS FromISR API variants from interrupt context.

A notification bit is not an event log. If you need a log or ordered sequence, use a queue or ring buffer.
