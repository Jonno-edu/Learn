Stateful Module Design

A compact reference for modules that remember state across calls without hiding ownership.

Default Shape

typedef struct {
    ModuleState state;
    uint32_t counter;
    bool fault_latched;
} Module;

void module_init(Module *module);
ModuleResult module_update(Module *module, const ModuleInput *input);

Rules

  • The caller owns context storage.
  • init writes every field to a known value.
  • The caller calls update once per new sample, event, or scheduler tick.
  • update is the normal mutation path and must preserve useful context between calls.
  • Inputs are usually borrowed as const *.
  • Private helpers and thresholds stay in the .c file.
  • Tests should create fresh contexts and replay sequences.

Caller Loop

Module module;
module_init(&module);

for each new input:
{
    ModuleResult result = module_update(&module, &input);
}

A hidden file-scope context is a review smell unless there is exactly one hardware instance and the ownership is documented.
