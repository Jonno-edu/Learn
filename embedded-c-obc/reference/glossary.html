Embedded C OBC Glossary

Short definitions for the terms introduced so far. This is the fast lookup page, not a full lesson.

Access And Hardware

volatile
A qualifier for objects that may change outside normal C code. Use for memory-mapped hardware registers. Not a synchronization primitive.
memory-mapped register
A hardware register exposed at an address that C code can read or write like memory.
register field
A subset of bits inside a register with one meaning, such as bit 3 meaning sensor measuring.
raw value
The direct decoded value from hardware before calibration, scaling, filtering, or unit conversion.

Bit Operations

bit
One binary digit, either 0 or 1.
nibble
Four bits. One hexadecimal digit.
byte
Eight bits. Usually two hexadecimal digits.
mask
A value used to isolate, set, clear, or test specific bits.
&
Bitwise AND. Keeps only bits that are 1 in both operands.
|
Bitwise OR. Combines bits, often to assemble non-overlapping fields.
<<
Left shift. Moves bits toward more significant positions.
>>
Right shift. Moves bits toward less significant positions.
whole-register equality
Comparing an entire register to one bit value. Usually wrong for status registers with independent bits.

C Expression Mechanics

operator precedence
The rules that decide expression grouping. Write (status & mask) == mask, not status & mask == mask.
cast
An explicit conversion such as (uint8_t)value. It should communicate intent.
narrowing
Converting from a wider type to a smaller type, such as uint32_t to uint8_t.
integer promotion
C's rule that small integer types are often promoted before arithmetic.
fixed-width integer
A type from <stdint.h> with explicit size, such as uint8_t or uint32_t.
two's complement
The signed integer representation used by modern C targets. A set top bit indicates a negative value.
size_t
An unsigned type used for object sizes and array lengths.
bounds check
A check that proves an index or copy length stays inside the available buffer before accessing memory.
checksum
A small computed value carried with a packet to catch corrupted or malformed data.
payload
The data part of a packet after headers and length fields.
frame
The complete on-wire packet bytes, including header, metadata, payload, and checksum.

RTOS Boundary

ISR
Interrupt service routine. Keep it short and avoid blocking work.
task
A schedulable FreeRTOS thread of execution. A task usually owns long-running module state.
blocked
A task state where the task waits for time, a queue item, notification, or other event.
tick
The kernel time unit used for delays, timeouts, and scheduling decisions.
task notification
A lightweight FreeRTOS mechanism for signaling a specific task.
queue
A FreeRTOS object for passing fixed-size items between tasks or from an ISR to a task.
QueueHandle_t
The opaque handle used to refer to a queue after creation.
queue by copy
Normal FreeRTOS queue behavior where item bytes are copied into queue storage and copied out into caller storage.
BaseType_t
A FreeRTOS integer type commonly used for pass/fail API return values.
TickType_t
A FreeRTOS integer type used for tick counts, delays, and queue block times.
pdPASS / pdFAIL
FreeRTOS-style success and failure values that must be checked before updating policy state.
portMAX_DELAY
A block-time value commonly used for indefinite waits when project configuration supports it.
FromISR API
A FreeRTOS API variant intended for interrupt context.
xQueueSendFromISR
The ISR-safe FreeRTOS queue-send API. It copies an item into a queue without blocking.
xHigherPriorityTaskWoken
A flag passed by address so a FromISR API can report that a higher-priority task was woken.
portYIELD_FROM_ISR
A port macro used at ISR exit to request a context switch when a higher-priority task was woken.
vTaskDelayUntil
A FreeRTOS task API for fixed-rate loops using a previous wake time plus a fixed tick increment.
periodic task
A task that runs work at a fixed cadence.
overrun
A condition where task work takes longer than its period budget.
freshness fault
A fault indicating that expected samples or updates stopped arriving within the allowed threshold.

Architecture And Process

responsibility boundary
The line around code that owns one reason to change, one state contract, or one policy.
execution boundary
The line around code that runs independently under the scheduler.
communication boundary
The line where data or signals cross between owners, such as a queue or notification.
integration boundary
The line around deployable or framework-wired software, such as a component or cFE application.
component
A framework-level software unit with explicit interfaces, commands, telemetry, events, and wiring.
port
A typed framework interface between components.
topology
The framework wiring that instantiates components and connects their ports into a deployed system.
cFE application
A deployable Core Flight Executive application integrated with cFE services, tasks, commands, telemetry, and software-bus messages.
requirement-driven test
A test derived from a stated behavior requirement and exercised through the public interface.
adapter
Code that connects a plain C policy module to FreeRTOS, hardware, or a flight-software framework boundary.
