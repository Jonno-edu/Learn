Embedded C OBC Glossary

Short definitions for the terms introduced so far. This is the fast lookup page, not a full lesson.

Access And Hardware

volatile
A qualifier for objects that may change outside normal C code. Use for memory-mapped hardware registers. Not a synchronization primitive.
memory-mapped register
A hardware register exposed at an address that C code can read or write like memory.
register field
A subset of bits inside a register with one meaning, such as bit 3 meaning sensor measuring.
raw value
The direct decoded value from hardware before calibration, scaling, filtering, or unit conversion.

Bit Operations

bit
One binary digit, either 0 or 1.
nibble
Four bits. One hexadecimal digit.
byte
Eight bits. Usually two hexadecimal digits.
mask
A value used to isolate, set, clear, or test specific bits.
&
Bitwise AND. Keeps only bits that are 1 in both operands.
|
Bitwise OR. Combines bits, often to assemble non-overlapping fields.
<<
Left shift. Moves bits toward more significant positions.
>>
Right shift. Moves bits toward less significant positions.
whole-register equality
Comparing an entire register to one bit value. Usually wrong for status registers with independent bits.

C Expression Mechanics

operator precedence
The rules that decide expression grouping. Write (status & mask) == mask, not status & mask == mask.
cast
An explicit conversion such as (uint8_t)value. It should communicate intent.
narrowing
Converting from a wider type to a smaller type, such as uint32_t to uint8_t.
integer promotion
C's rule that small integer types are often promoted before arithmetic.
fixed-width integer
A type from <stdint.h> with explicit size, such as uint8_t or uint32_t.
two's complement
The signed integer representation used by modern C targets. A set top bit indicates a negative value.
size_t
An unsigned type used for object sizes and array lengths.
bounds check
A check that proves an index or copy length stays inside the available buffer before accessing memory.
checksum
A small computed value carried with a packet to catch corrupted or malformed data.
payload
The data part of a packet after headers and length fields.
frame
The complete on-wire packet bytes, including header, metadata, payload, and checksum.
