OBC Boundary Classification

Use this table before deciding whether something should be a plain C module, FreeRTOS task, queue, or larger app/component.

BoundaryAnswersTypical artifact
ModuleWho owns this responsibility and state?.h/.c with public API, private helpers, unit tests.
TaskWhat runs independently under the scheduler?Task function, priority, stack, blocking points.
Queue or notificationWhat crosses between execution owners?Queue item type, notification bits, timeout policy.
App/componentWhat is deployed, wired, commanded, and integrated?F Prime component/topology, cFE app, service integration.

Classification Prompt

  1. Write the responsibility in one sentence.
  2. State whether it needs its own scheduler loop.
  3. State what data or events cross into or out of it.
  4. State whether the mission framework needs to deploy or command it directly.
  5. Only then choose the artifact.

A single feature can require several boundaries. Keep them named separately instead of making one file do everything.
