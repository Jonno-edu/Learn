Mini OBC Integration Slice Checklist

A compact process for building a reviewable sensor-to-flight-task feature.

Build Order

  1. State the OBC problem in one sentence.
  2. Choose the plain C module boundary.
  3. Write public types and function prototypes.
  4. Derive tests from requirements.
  5. Implement one behavior at a time.
  6. Add the FreeRTOS adapter after the core behavior is testable.
  7. Review with a safety-critical C checklist.

Requirement To Test Map

Requirement typeTest shape
InitializationCall init and check every public field with expected defaults.
Nominal updateFeed valid samples and check counters, last tick, and result codes.
Drop or stale behaviorCall failure recorders until the threshold and check fault latches.
Invalid inputPass NULL and assert a fault result without a crash.
Adapter behaviorCheck every FreeRTOS return code before mutating health state.

Review Checklist

  • No recursion, hidden unbounded loops, or task delays inside policy modules.
  • All pointer arguments are checked before first dereference.
  • All queue return values are checked.
  • Fault latches are explicit and tested.
  • Public headers expose contracts, not private helper details.
  • Host tests compile warning-clean.

Do not start the integration slice by wiring every task. Start with the smallest plain C contract that proves the health behavior.
