Event-Driven State Machines

A reference for flight logic that transitions on named events.

Default Shape

void fsm_init(Fsm *fsm);
FlightPhase fsm_dispatch(Fsm *fsm, FlightEvent event);

switch (fsm->phase)
{
case FLIGHT_PHASE_IDLE:
    return dispatch_idle(fsm, event);
case FLIGHT_PHASE_BOOST:
    return dispatch_boost(fsm, event);
default:
    fsm->phase = FLIGHT_PHASE_FAULT;
    return fsm->phase;
}

Rules

  • Convert raw observations into named events before transition policy when practical.
  • Dispatch one event at a time.
  • Switch on current state first.
  • Keep per-state transition helpers private.
  • Decide whether unexpected events are ignored, logged, or faults.
  • Keep RTOS delivery outside the FSM core so the FSM remains host-testable.

Do not let every task mutate the FSM context. In an RTOS design, one task should usually own the FSM and receive events.
