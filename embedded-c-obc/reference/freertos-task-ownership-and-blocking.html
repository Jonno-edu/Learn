FreeRTOS Task Ownership And Blocking

A reference for shaping task loops around clear ownership and blocking behavior.

Default Shape

static void task(void *argument)
{
    TaskContext *ctx = argument;

    for (;;)
    {
        Event event;
        if (wait_for_event(&event, ctx->timeout_ticks))
        {
            dispatch_owned_state(ctx, event);
        }
        else
        {
            handle_timeout(ctx);
        }
    }
}

Rules

  • Give each task a clear ownership reason to exist.
  • Prefer blocking on work over busy polling.
  • Document whether helper APIs can block.
  • Make timeout behavior explicit and testable.
  • Keep one owner for long-lived state such as the flight FSM context.

A task that spins without delay or blocking is usually stealing CPU time from more useful work.
