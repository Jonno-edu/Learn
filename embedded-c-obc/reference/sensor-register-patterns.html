Sensor Register Patterns

Real sensor drivers are mostly careful translation from a datasheet register map into named masks, shifts, and byte-combining helpers. Treat the datasheet as the contract and the C code as a direct, reviewable encoding of that contract.

Status Bits

A status register often contains independent bits. For a BMP280-style status byte, bit 3 indicates a measurement is running and bit 0 indicates image-register update is running. 

enum {
    BMP280_STATUS_MEASURING = 1u << 3,
    BMP280_STATUS_IM_UPDATE = 1u << 0
};

bool bmp280_is_measuring(uint8_t status)
{
    return (status & BMP280_STATUS_MEASURING) == BMP280_STATUS_MEASURING;
}

Combining Packed Bytes

A 20-bit pressure value can be stored across three 8-bit registers: MSB, LSB, and the upper nibble of XLSB. Shift each byte into its final position, mask when a byte contains unused bits, then OR the fields together. 

uint32_t raw_pressure =
    ((uint32_t)msb << 12) |
    ((uint32_t)lsb << 4) |
    ((uint32_t)xlsb >> 4);

Cast before shifting when the source is a byte but the result is wider. This prevents the expression from depending on small-type promotion details and makes the intended 32-bit assembly obvious.

Review Checklist

  • Do mask names match datasheet field names?
  • Does the function ignore unrelated bits?
  • Are bytes cast to the output width before large shifts?
  • Are unused bits masked or shifted away before combining?
  • Do tests include unrelated status bits and nonzero unused bits?
