Lesson 0010: Event-Driven State Machines

A state machine should not be a pile of sensor thresholds. Convert observations into named events, then let each state decide which events it accepts.

The One Thing

An event-driven state machine separates detection from transition policy. Detection says "apogee detected." The FSM says "from coast, apogee moves to descent."

OBC habit: state transitions should read like a flight-review checklist: current state, incoming event, next state.

Event API Shape

typedef enum {
    FLIGHT_EVENT_NONE = 0,
    FLIGHT_EVENT_LAUNCH_DETECTED,
    FLIGHT_EVENT_BURNOUT_DETECTED,
    FLIGHT_EVENT_APOGEE_DETECTED,
    FLIGHT_EVENT_LANDING_DETECTED,
    FLIGHT_EVENT_SENSOR_FAULT
} FlightEvent;

FlightPhase flight_fsm_dispatch(FlightFsm *fsm, FlightEvent event);

The caller or a detector module can produce events. The FSM consumes one event at a time and owns the transition rules.

Switch By Current State

The usual embedded-C layout is a top-level switch on current state, then a smaller switch or if-chain for accepted events.

switch (fsm->phase)
{
case FLIGHT_PHASE_IDLE:
    return update_idle(fsm, event);

case FLIGHT_PHASE_BOOST:
    return update_boost(fsm, event);

default:
    fsm->phase = FLIGHT_PHASE_FAULT;
    return fsm->phase;
}

This keeps each state's transition policy small and reviewable.

Ignore, Reject, Or Fault?

Unexpected eventCommon response
Benign duplicate, such as another LAUNCH_DETECTED while already in boostIgnore and stay in current state.
Impossible but non-dangerous ordering, such as landing while idleUsually ignore or log in real firmware.
Sensor fault, corrupt state, invalid enumLatch or enter fault depending on system design.
Design point: "unexpected" does not always mean "fault." Decide that policy deliberately and test it.

FreeRTOS Preview

FreeRTOS fits after this boundary. A task can own the FSM context, receive events from a queue or notification, then dispatch them one at a time. The FSM itself stays ordinary testable C.

for (;;)
{
    FlightEvent event = wait_for_next_event();
    (void)flight_fsm_dispatch(&fsm, event);
}
