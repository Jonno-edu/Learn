Lesson 0012: FreeRTOS Task Ownership And Blocking

A FreeRTOS task is not just a function with a loop. It owns stack, time, blocking behavior, and usually a piece of long-lived module state.

The One Thing

A well-shaped task owns one responsibility and spends most of its life blocked waiting for work, rather than polling in a tight loop.

OBC habit: one flight-control task can own the FSM context. Other code sends it events; it wakes, dispatches one event, then blocks again.

Task Shape

static void flight_task(void *argument)
{
    FlightTaskContext *ctx = argument;

    for (;;)
    {
        FlightEvent event;
        if (wait_for_event(&event, ctx->wait_timeout_ticks))
        {
            (void)flight_fsm_dispatch(&ctx->fsm, event);
        }
        else
        {
            record_idle_timeout(ctx);
        }
    }
}

The infinite loop is normal. The important part is that wait_for_event blocks, so the task gives CPU time back to the scheduler while there is no work.

Ownership Checklist

QuestionGood answerReview smell
What state does this task own?The flight FSM context and its counters.Many tasks write the same state directly.
What does it wait on?An event source with a clear timeout.A tight loop checking a flag forever.
What happens on timeout?A deliberate idle/health/check path.The timeout case is ignored accidentally.
Can it block inside shared code?Only if the API contract says so.A helper unexpectedly sleeps or waits.

Blocked Is Healthy

In FreeRTOS, a blocked task is waiting for something: time to pass, a queue item, a notification, a semaphore, or another kernel event. That is usually healthier than burning CPU cycles checking the same condition repeatedly.

Review smell: for (;;) { if (flag) { ... } } with no delay or blocking call is usually a busy poll.

Exercise Model

The exercise does not include FreeRTOS headers. It gives you a host-side model of one task step: receive an event if one is pending, dispatch later, or record an idle timeout.

FlightTaskResult result =
    flight_task_step(&task, &source, &event_to_dispatch);

if (result == FLIGHT_TASK_DISPATCH_EVENT)
{
    (void)flight_fsm_dispatch(&fsm, event_to_dispatch);
}

This is the same shape as a task that blocks on a queue or notification, but small enough to test locally.
