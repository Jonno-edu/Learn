Lesson 0006: Embedded C Toolbox

Firmware C gets easier when every declaration answers one question: is this a byte of data, a count, an address, a hardware-visible object, or a shared/concurrent value?

The One Thing

Pick C types by the job they do in the firmware, not by habit. A byte from a packet is different from a buffer length. A memory-mapped register is different from a normal variable. A shared flag is different from a hardware register.

OBC habit: when you declare a variable, say out loud whether it is data, a size, an address, a hardware access, or a synchronization boundary.

The Core Toolbox

Tool Use it for Firmware example
uint8_t One unsigned byte of actual data. Packet byte, register byte, checksum, payload element.
uint16_t, uint32_t Known-width unsigned fields and registers. ADC count, packed sensor field, 32-bit status register.
int16_t, int32_t Signed sensor values and wider signed math. Accelerometer raw reading, scaled intermediate calculation.
size_t Object sizes, buffer lengths, and array indexes. frame_len, payload_index, loop counter over bytes.
bool A true/false result, state, or predicate. sensor_ready, parse_ok, armed.
Pointer Where an object or buffer begins. const uint8_t *frame points at incoming bytes.
Array Actual storage for a fixed number of elements. uint8_t payload[8] stores copied payload bytes.
struct A named group of related fields. ObcPacket groups message ID, length, and payload.
enum Named integer constants. Flight modes, parser states, command IDs, fixed protocol constants.

Qualifiers And Storage Words

const

Read-only through this name. Use it for lookup tables, protocol constants, and input buffers the function must not mutate. 

bool parse_frame(const uint8_t *frame, size_t frame_len);

volatile

Every access matters to the compiler. Use for memory-mapped hardware registers. Do not use it as a mutex, atomic, or FreeRTOS synchronization replacement. 

volatile uint32_t * const status_reg = REG_STATUS;

static

At file scope, hides a function or object from other translation units. Inside a function, gives an object static lifetime. In firmware, prefer file-scope static helpers when they are private to a module. 

static uint8_t checksum(const uint8_t *bytes, size_t len);

_Atomic / atomic_*

C11 atomic objects avoid data races between threads. In RTOS firmware, confirm toolchain and platform support before using them instead of the RTOS primitives. 

#include <stdatomic.h>
atomic_bool telemetry_ready;

restrict

A promise that a modified object is accessed only through that pointer path. Powerful for optimization, risky if you cannot prove the no-alias contract. 

void copy(size_t n, uint8_t * restrict dst,
          const uint8_t * restrict src);

NULL

A pointer value meaning "points at no object." Check it before dereferencing API inputs from callers, drivers, or test harnesses. 

if (frame == NULL || out == NULL) {
    return false;
}

Three Declarations From The Parser

static bool obc_parse_frame(const uint8_t *frame,
                            size_t frame_len,
                            ObcPacket *out);
Part Meaning
const uint8_t *frame The caller gives us a starting address for read-only bytes.
size_t frame_len The caller tells us how many bytes are valid from that starting address.
ObcPacket *out The caller gives us an address where we may write the parsed result after validation.
Parser trap: inside a function, an array parameter is already pointer-like. Do not use sizeof(frame) to discover the original caller's array length. Carry the length in a separate size_t parameter.

Review Drill

Open your Lesson 5 parser and read every declaration. For each one, label it with one of these: 

data byte | size/index | address | output storage | hardware access | true/false result

If a declaration cannot be labeled clearly, ask me about it. That is usually where C confusion starts.
