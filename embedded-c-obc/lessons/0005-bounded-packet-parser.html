Lesson 0005: Bounded Packet Parser

Comms code is where byte-level C meets reliability. This lesson turns your mask/shift muscle into a defensive parser for a small OBC-style binary frame.

The Frame Contract

The frame is a tiny training protocol:

Offset Field Example
0start byte 00xa5
1start byte 10x5a
2message ID0x10
3payload length0x03
4..6payload0x01 0x02 0x03
7checksum0x19

Checksum is the low byte of the sum of msg_id, payload_len, and every payload byte. For the example: 0x10 + 0x03 + 0x01 + 0x02 + 0x03 = 0x19.

The One Thing

Before reading frame[i], prove i is inside the buffer. Before copying payload bytes, prove the payload length fits both the input frame and the output packet.

Parser habit: validate structure first, write output last. A rejected frame should not leave a half-updated packet behind.

Bounds Before Bytes

if (frame_len < MIN_FRAME_SIZE) {
    return false;
}

if (frame[0] != SOF0 || frame[1] != SOF1) {
    return false;
}

The minimum length check comes before reading frame[0] or frame[1]. That is the pattern to repeat throughout the parser.
