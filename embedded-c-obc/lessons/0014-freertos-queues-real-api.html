Lesson 0014: FreeRTOS Queues Real API

This is the shift from queue concepts to real FreeRTOS queue muscle memory: create a queue, send copied IMU samples, block the flight task on receive, and check every return value.

The Embedded Problem

An IMU sensor task produces samples at a fixed rate. A flight task consumes those samples in order. The tasks should not share a writable global sample struct.

OBC habit: the sensor task owns sampling. The flight task owns processing. The queue is the boundary between them.

Module Boundary

The queue carries one fixed-size item type. In this lesson that item is an ImuSample.

typedef struct
{
    uint32_t sequence;
    int16_t accel_mg[3];
    int16_t gyro_mdps[3];
    TickType_t sampled_at_tick;
} ImuSample;

typedef struct
{
    QueueHandle_t sample_queue;
    uint32_t sent_sample_count;
    uint32_t received_sample_count;
    uint32_t dropped_sample_count;
} ImuQueueApp;

The public boundary exposes a handle, counters, and functions that make the return-value policy visible.

The API To Type

SymbolWhat it means in this exercise
QueueHandle_tThe opaque handle returned by queue creation and passed to send/receive.
xQueueCreateCreate storage for a fixed queue depth and fixed item size.
xQueueSendCopy one item into the queue from task context.
xQueueReceiveCopy one item out of the queue into caller-provided storage.
BaseType_tThe FreeRTOS return type used for pass/fail style results.
TickType_tThe FreeRTOS tick-count type used for block times.
pdPASS / pdFAILReturn values you must check instead of assuming success.
portMAX_DELAYA common "wait indefinitely" block value when the project configuration supports it.

Queue By Copy

FreeRTOS queues copy bytes. The queue does not magically take ownership of local stack objects unless you deliberately queue pointers.

ImuSample sample = read_imu_sample();
BaseType_t sent = xQueueSend(queue, &sample, (TickType_t)0);

ImuSample received;
BaseType_t got = xQueueReceive(queue, &received, portMAX_DELAY);
Plain C still matters: &sample is the address of the item to copy into the queue. &received is the address of storage to copy out into.

Producer Task Shape

for (;;)
{
    ImuSample sample = read_imu_sample();

    if (xQueueSend(app->sample_queue, &sample, (TickType_t)0) == pdPASS)
    {
        app->sent_sample_count++;
    }
    else
    {
        app->dropped_sample_count++;
    }

    vTaskDelay(pdMS_TO_TICKS(IMU_SAMPLE_PERIOD_MS));
}

The zero tick block time means fail fast if the queue is full. That makes overflow visible.

Flight Task Shape

for (;;)
{
    ImuSample received;

    if (xQueueReceive(app->sample_queue, &received, portMAX_DELAY) == pdPASS)
    {
        app->received_sample_count++;
        process_imu_sample(&received);
    }
}

The flight task blocks while the queue is empty. It wakes when a sample is available, receives the oldest queued item, and processes samples in order.
