Lesson 0004: Signed Sensor Values

Many sensors return signed two-byte values. The skill is to combine bytes as an unsigned pattern, then interpret that exact 16-bit pattern as a signed value, then scale using a wide enough type.

The One Thing

Byte assembly and signed interpretation are separate steps. First build the 16-bit pattern. Then convert that pattern into the signed sensor count. 

uint16_t raw = ((uint16_t)msb << 8) | (uint16_t)lsb;
int16_t counts = (int16_t)raw;
Driver habit: preserve the exact bit pattern first. Apply signed meaning after the full-width raw pattern exists.

Why 0xff38 Is Negative

In a signed 16-bit two's-complement value, bit 15 is the sign bit. If it is set, the value is negative.

Bytes 16-bit pattern Signed meaning
0x00 0x64 0x0064 100
0xff 0x38 0xff38 -200
0x80 0x00 0x8000 -32768

Scaling Without Overflow

If one count equals 61 milli-g, multiply in a wider signed type before narrowing or returning: 

int32_t milli_g = (int32_t)counts * 61;

This avoids doing the multiplication in a type that may be too small and tells reviewers the intermediate range was considered.
