Lesson 0017: Periodic Sensor Task Timing

A fixed-rate sensor task should wake on a schedule, do bounded work, make send failures visible, and then return to the scheduler.

The Embedded Problem

The IMU producer must sample at a predictable cadence. A loose vTaskDelay(period) after the work makes the loop period depend on how long the work took. vTaskDelayUntil keeps the task aligned to a previous wake time plus a fixed increment.

OBC habit: separate timing cadence from health policy. The task owns the loop; a small module owns stale/drop counters.

Loop Shape

void imu_sensor_task(void *argument)
{
    ImuProducer *producer = argument;
    TickType_t last_wake = xTaskGetTickCount();
    const TickType_t period = pdMS_TO_TICKS(10u);

    for (;;)
    {
        ImuSample sample = imu_read_sample();
        BaseType_t sent = xQueueSend(producer->queue, &sample, (TickType_t)0);
        imu_timing_policy_record_send(&producer->timing, sent == pdPASS);

        vTaskDelayUntil(&last_wake, period);
    }
}

The task does not hide an infinite retry loop inside the producer. If the queue is full, the sample is dropped and the health policy records it.

Timing Review Table

QuestionDesign answer
Who owns the loop?The FreeRTOS task function.
Who owns the sample policy?A plain C module with counters and thresholds.
What happens when the queue is full?The producer records a missed delivery instead of blocking forever.
What is an overrun smell?The work takes so long that the next release time is already due.
What should telemetry expose?Drop counts, consecutive missed sends, last sample tick, and any freshness fault.

Do Not Hide The Scheduler

The sensor driver can provide imu_read_sample(). The queue adapter can provide imu_queue_send(). The task is still the place that repeats forever and calls vTaskDelayUntil.

Review smell: a plain C module that calls vTaskDelayUntil internally is no longer an ordinary policy module. It has become an execution boundary.

Quick Check

2. What should the timing policy module avoid owning?
