Lesson 0003: A Real Sensor Register Slice

This is the same C muscle as lesson 2, but in a shape you would actually meet in a pressure-sensor driver: status bits plus a packed 20-bit ADC sample spread across three bytes.

The Datasheet Shape

A BMP280-style pressure sensor has a status register where bit 3 means a conversion is running and bit 0 means internal image-register update is running. These are independent bits, so whole-register equality is the wrong habit. 

enum {
    BMP280_STATUS_MEASURING = 1u << 3,
    BMP280_STATUS_IM_UPDATE = 1u << 0
};
Real-driver habit: name the datasheet bit, mask the register, compare the masked result.

The Packed Measurement Shape

The raw pressure reading is a 20-bit value. It arrives as three bytes:

  • press_msb: pressure bits 19..12
  • press_lsb: pressure bits 11..4
  • press_xlsb: pressure bits 3..0 live in the upper nibble

So the assembly pattern is: 

((uint32_t)press_msb << 12) |
((uint32_t)press_lsb << 4)  |
((uint32_t)press_xlsb >> 4)

The third byte is shifted right because only its upper four bits belong to the 20-bit sample.

Visual Example: 0x12 0x34 0x50

The test case bmp280_raw_pressure(0x12, 0x34, 0x50) should produce 0x12345. Think of the final 20-bit value as five hex digits. Each hex digit is four bits.

press_msb = 0x12 ((uint32_t)press_msb << 12) 0x12000 Moves the most significant byte into bits 19..12.
press_lsb = 0x34 ((uint32_t)press_lsb << 4) 0x00340 Moves the next byte into bits 11..4.
press_xlsb = 0x50 ((uint32_t)press_xlsb >> 4) 0x00005 Drops the unused lower nibble and moves the upper nibble into bits 3..0.
combined with | 0x12000 | 0x00340 | 0x00005 0x12345 OR joins non-overlapping fields into one raw sample.
No clamping in this helper: BMP280_RAW_PRESSURE_MAX is the largest possible 20-bit value, 0xfffff. The decoder should return the assembled raw value. If all input bytes are 0xff, the assembled value naturally equals the max because 0xfff... fills every 20-bit field position.
