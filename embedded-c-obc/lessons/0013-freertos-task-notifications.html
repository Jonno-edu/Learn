Lesson 0013: FreeRTOS Task Notifications

Task notifications are a lightweight way to wake or signal one specific task. Use them when the receiver is known and the payload can be compact.

The One Thing

A task notification is a per-task signal slot. For an OBC flight task, it can wake the owner task when a compact event bit is set.

OBC habit: use a notification for "wake this task for these event bits"; use a queue when you need to preserve a sequence of event objects.

Notification Bits

One common pattern is to treat the notification value as a bitfield. Producers set bits; the task reads and clears the bits it handles.

#define FLIGHT_NOTIFY_LAUNCH   (1u << 0)
#define FLIGHT_NOTIFY_BURNOUT  (1u << 1)
#define FLIGHT_NOTIFY_APOGEE   (1u << 2)
#define FLIGHT_NOTIFY_FAULT    (1u << 3)

/* Later FreeRTOS shape. */
xTaskNotifyFromISR(flight_task_handle,
                   FLIGHT_NOTIFY_APOGEE,
                   eSetBits,
                   &higher_priority_task_woken);

Notification Or Queue?

NeedUsually chooseReason
Wake one known taskTask notificationLow overhead and direct.
Send compact flagsTask notification bitsBits can coalesce safely when each bit means "work pending".
Preserve every event in orderQueueNotifications can merge repeated identical bits.
Send structs or samplesQueueA queue carries items, not just a notification value.
Design point: if two apogee notifications arrive before the task wakes, a bitfield still only says "apogee bit is set." That is fine for some events and wrong for others.

Task-Side Shape

for (;;)
{
    uint32_t bits = wait_for_notification_bits();

    if ((bits & FLIGHT_NOTIFY_APOGEE) != 0u)
    {
        (void)flight_fsm_dispatch(&fsm, FLIGHT_EVENT_APOGEE_DETECTED);
    }

    if ((bits & FLIGHT_NOTIFY_FAULT) != 0u)
    {
        (void)flight_fsm_dispatch(&fsm, FLIGHT_EVENT_SENSOR_FAULT);
    }
}

The task still owns the FSM. The notification only wakes it and tells it which compact events are pending.
