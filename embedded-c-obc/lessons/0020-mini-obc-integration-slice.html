Lesson 0020: Mini OBC Integration Slice

This is the first consolidation project: design a small IMU-to-flight-task slice before filling in the implementation.

The Slice

Build the smallest reviewable path from a sensor producer to health-visible flight logic.

  1. Sensor task produces IMU samples at a fixed cadence.
  2. Queue carries samples to the flight task.
  3. Plain C module records drops, stale receives, and last good sample timing.
  4. Health status exposes overload and freshness faults.
  5. FreeRTOS adapter owns task and queue mechanics.
Do this in order: problem, boundary, API, tests, implementation, review.

Requirements First

IDRequirementTest idea
REQ-001Initialize health state with explicit thresholds.Check all counters, ticks, and latches after init.
REQ-002Record each queued sample and remember the latest sequence/tick.Feed two samples and check latest values.
REQ-003Latch overload fault after repeated queue drops.Call drop recorder until threshold is reached.
REQ-004Latch freshness fault after repeated empty receives or missed deliveries.Call stale recorder until threshold is reached.
REQ-005Invalid pointers return a fault result without dereferencing.Pass NULL to every public function that accepts a pointer.

Boundary Sketch

/* Plain C policy module. */
ImuHealthResult imu_health_record_queued_sample(ImuHealth *health,
                                                uint32_t sequence,
                                                uint32_t now_tick);
ImuHealthResult imu_health_record_queue_drop(ImuHealth *health);
ImuHealthResult imu_health_record_empty_receive(ImuHealth *health);

/* FreeRTOS adapter layer, later. */
sensor_task --xQueueSend--> sample_queue --xQueueReceive--> flight_task
flight_task --calls--> imu_health module

The exercise starts with the plain C health module. The FreeRTOS task and queue shape is documented, but the first implementation target is host-testable.
