Lesson 0019: Module, Task, Queue, App Boundaries

Before writing code, name the kind of boundary. A module, task, queue, and application are different answers to different design questions.

The Four Boundaries

BoundaryQuestion it answersOBC example
ModuleWhat responsibility and state does this code own?Recovery arming policy, IMU freshness policy, telemetry packing.
TaskWhat runs independently under the scheduler?Sensor producer, flight event consumer, telemetry downlink task.
Queue or notificationWhat crosses between execution owners?IMU sample queue, one-task wakeup bit, command message stream.
App or componentWhat is deployed, wired, commanded, and integrated as a larger unit?cFE app, F Prime component, mission-specific flight software component.
Central rule: a module is a responsibility boundary; a task is an execution boundary; a queue is a communication boundary; an app/component is an integration boundary.

Architecture Sources Agree On The Shape

F Prime separates components, ports, and topology. cFE separates applications, tasks, service APIs, and messages. Those frameworks are larger than this lesson, but the useful habit is small: define the interface before mixing responsibility, scheduling, and deployment together.

Review smell: a file named like a module but containing an infinite task loop, queue creation, hardware reads, and state-machine policy probably has too many boundary types in one place.

Example Split

/* Plain C module. */
ImuHealthResult imu_health_record_sample(ImuHealth *health,
                                         const ImuSample *sample);

/* FreeRTOS adapter task. */
void imu_flight_task(void *argument)
{
    FlightRuntime *runtime = argument;

    for (;;)
    {
        ImuSample sample;
        if (xQueueReceive(runtime->sample_queue, &sample, portMAX_DELAY) == pdPASS)
        {
            (void)imu_health_record_sample(&runtime->imu_health, &sample);
        }
    }
}

The health module can be unit-tested without FreeRTOS. The task can be reviewed for scheduler behavior. The queue is just the communication boundary between them.
